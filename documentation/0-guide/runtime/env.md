# 运行环境

AIUI 智能体运行在多种平台上：Rokid 穿戴式设备（如 Rokid Glasses 等）、桌面端调试器以及未来即将支持的更多 AI + AR 硬件。

不同运行环境下，脚本执行环境以及用于组件渲染的环境虽然在设计上保持统一，但在底层实现和性能表现上存在一定差异：

| 运行环境 | 底层实现与特点 |
|:--|:--|
| **Rokid Glasses（乐奇眼镜）** | 逻辑层和视图层运行在基于 **QuickJS** 和 **Skia** 构建的高性能自研容器（Ink）中，针对穿戴式设备的极低功耗、内存和 CPU 限制进行深度优化。 |
| **Craft：一款基于 AIUI 的 Web IDE** | 基于 Web 的 AIUI 开发环境，支持代码编辑、工程管理、智能体绑定与实时预览，方便开发者在线完成智能体的编写与调试。 |
| **Rokid AR Lite / AR Studio**（计划中） | 未来将通过适配底层运行环境，实现智能体跨不同 AI + AR 设备的一致性体验。 |

## 平台差异

尽管 AIUI 致力于在各运行环境中提供一致的开发体验，但实际运行中仍然会有一些区别：

1. **硬件与传感器差异**：在 DevTools 模拟器中，部分依赖真实物理环境的 API（如 6DoF 空间追踪、高精度手势识别、空间音频以及特定传感器数据）通常只能通过模拟数据返回。开发者必须在真实的 Rokid 设备上验证这些交互的真实表现。
2. **性能表现差异**：桌面端（DevTools）拥有充裕的计算资源，而穿戴式设备对功耗和发热有极其严格的限制。因此，在桌面端运行流畅的复杂动画或高频数据更新（如频繁的 `setData`），在真实设备上可能会触发性能瓶颈或系统降频策略。
3. **渲染与排版表现**：虽然底层都通过一致的响应式管线描述视图，但在不同分辨率、不同 FOV（视场角）的物理设备上，实际的空间视觉比例、深度感知和排版表现可能会有细微差异。

> **注意**：开发者工具（AIUI DevTools）仅供开发和调试使用，智能体的最终 UI 表现、交互体验和性能指标请务必以 Rokid 客户端真机为准。

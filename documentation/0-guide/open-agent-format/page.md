# 页面概览

在 AIUI 对 Open Agent Format 的扩展里，`pages/` 目录用来定义智能体的具体界面与交互页面。页面不仅承载 UI 结构，也承载页面级数据、生命周期回调和事件处理逻辑。

如果说：

- `AGENTS.md` 定义的是智能体是谁
- `app.json` 定义的是应用如何启动

那么“页面”定义的就是：用户实际会看到什么，以及如何与智能体交互。

## 页面组织方式

在 AIUI 中，页面通常有两种组织方式：

- **多文件页面**：页面由 `index.js`、`index.wxml`、`index.wxss` 和 `page.json` 等文件共同组成，其中页面导出对象通常写在 `index.js` 中
- **单文件页面**：页面使用 `.ink` 单文件组织，在同一个文件中同时声明配置、逻辑、结构和样式；其中页面逻辑通常写在 `<script setup>` 中

你可以把它理解成：

- 多文件页面：适合按“配置 / 逻辑 / 结构 / 样式”拆分维护
- 单文件页面：适合把同一个页面的内容放在一起阅读和编辑

无论采用哪种组织方式，页面本身的生命周期、`data` 数据对象和事件处理方式保持一致。

## 启动环境

除了页面文件如何组织，页面还会以不同的启动环境被使用。在 AIUI 中，目前常见有两种页面承载方式：

- **聊天窗口内使用**：页面嵌入在聊天窗口中，适合在对话流里补充更丰富、更有层次的信息展示
- **独立窗口 / modal 使用**：页面在新的窗口或 `modal` 中打开，适合更完整、更沉浸式的交互流程

这里需要注意，`target` 并不是由开发者直接在页面里手动声明的字段，而是 AIUI 根据开发者为每个页面提供的配置和使用意图来决定页面最终如何启动。

你可以把它理解成：

- `_current`：不离开当前聊天上下文，在消息流内完成展示和交互
- `_blank`：把页面作为一个独立承载空间来使用，强调沉浸式体验

一般来说，当页面先以聊天窗口内的方式使用时，用户后续仍然可以通过点击把它切换到 `_blank` 模式，以获得更大的展示空间和更完整的交互体验。

如果你的页面主要是结果展示、卡片补充、状态回填，通常会先以聊天窗口内的方式承载。  
如果你的页面承载的是完整流程、表单、多步骤操作或更强交互，则更适合进入独立窗口或 `modal`。

## 在 Open Agent Format 里的位置

从 Open Agent Format 视角看，页面是 AIUI 把智能体能力变成可交互界面的关键一层：

- `AGENTS.md`：定义智能体身份和系统指令
- `app.json`：定义应用入口和全局配置
- `pages/`：定义具体页面以及界面交互

因此，AIUI 对 OAF 的扩展重点之一，就是把原本只停留在描述层的智能体，扩展成真正可运行、可交互的 AI-Native User Interface。

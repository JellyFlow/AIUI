# Open Agent Format

Open Agent Format，简称 OAF，是一套用于描述智能体组成方式的开放规范。它强调用目录和文件来表达智能体，让智能体的身份、说明、能力边界和可组合资源可以被清晰组织、读取和迁移。

AIUI 采用并扩展了这一思路：在兼容 `AGENTS.md` 这一核心描述文件的基础上，进一步补充了界面和运行组织方式，让一个智能体不仅能被描述，也能被运行和展示。

## 它解决什么问题

如果没有统一的工程结构，不同平台往往会用各自的方式描述智能体：

- 智能体是谁
- 它具备哪些能力
- 它依赖哪些资源
- 它如何被调起
- 它如何以界面形态与用户交互

OAF 想解决的，就是这些信息的组织和可移植性问题。它希望把“一个智能体是什么”尽量落回到目录和文件本身，而不是散落在平台私有配置里。

## 在 AIUI 里怎么理解

在 AIUI 中，你可以把 Open Agent Format 理解成“智能体的描述层”，而 AIUI 在这个基础上补上了“应用和界面层”。

一个典型的 AIUI 智能体通常包含三类关键内容：

- `AGENTS.md`：定义智能体是谁、做什么、遵循什么指令
- `app.json`：定义应用入口和全局配置
- `pages/`：定义页面组织方式和用户交互界面

你也可以把它简单理解成：

- **OAF 负责描述智能体**
- **AIUI 负责把智能体变成可交互的 AI-Native User Interface**

## 为什么它重要

对于开发者来说，OAF 的价值不在于增加一套新的概念，而在于让智能体工程更容易理解、维护和迁移：

- 目录结构更清晰
- 描述文件更集中
- 工程更容易在不同平台之间映射
- 界面层和智能体描述层的边界更明确

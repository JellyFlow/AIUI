# 介绍

AIUI 智能体框架的核心，不只是“把页面渲染出来”，而是把智能体描述、页面入口、逻辑层和视图层组织成一个完整的运行体系。开发者负责定义 agent 的身份、页面结构和业务逻辑，运行时再把这些内容串起来，让状态变化持续映射成界面反馈。

下面这个交互式说明会从两个角度帮助你理解 AIUI 智能体框架：

- 它由哪些部分组成
- 一个页面更新在运行时是怎么发生的

## 核心概念

AIUI 采用经典的逻辑层与视图层分离设计，但在智能体框架里，这个分层并不是孤立存在的，而是和 agent 描述、应用入口、页面结构一起协同工作：

1. **逻辑层 (Logic Layer)**：运行在 QuickJS 中，负责处理业务逻辑、API 调用和数据状态管理。
2. **视图层 (View Layer)**：由 WXML 和 WXSS 组成，运行在 Ink 渲染引擎中，负责页面的结构与样式呈现。
3. **应用入口 (Application Entry)**：由 `app.json` 定义页面入口和全局窗口配置，决定 agent 启动后先进入哪个页面。
4. **智能体描述 (Agent Description)**：由 `AGENTS.md` 定义智能体的身份、描述、能力和系统指令，决定平台如何理解和调度这个 agent。

这种架构不仅能提供流畅的界面更新能力，也让开发者能够把“智能体是谁”“入口从哪里开始”“逻辑怎么流转”“界面如何反馈”拆开组织，更适合 AI + AR 这种高频交互和持续状态更新的场景。

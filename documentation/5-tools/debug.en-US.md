# On-Device Debugging

On-device debugging is a critical step in verifying the real user experience of AIUI agents. Compared with desktop or simulated environments, real devices can more accurately reflect device-side performance, interaction feedback, network conditions, and local capability invocation.

## Applicable Scenarios

- **Interaction Verification**: Confirm the actual behavior of voice, gestures, buttons, and other interactions on the device.
- **Performance Observation**: Troubleshoot issues related to startup time, stuttering, memory usage, and power consumption.
- **Capability Integration Testing**: Verify the actual behavior of network, storage, audio, and other local capabilities on the device.

## Debugging Recommendations

- Prioritize real-device results as the final basis for judging the user experience.
- Use log output and remote debugging capabilities together to locate runtime issues.
- Repeatedly verify weak-network, Bluetooth link, and continuous-operation scenarios in key task flows.

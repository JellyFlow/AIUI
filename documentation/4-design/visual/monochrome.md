# Monochrome

`Monochrome` 用于描述 AIUI 在单色显示设备上的设计语言。

这类设备只能在纯黑背景上渲染单一发光通道，因此不能依赖多色语义来表达层级、警告、高亮或数据差异，而是需要把层级表达收敛到透明度、描边、字重、间距和版式结构上。

## 适用设备

| 变体 | 目标设备 | 状态 |
| --- | --- | --- |
| `Green` | RokidGlasses1 / RokidGlasses2 | 已启用 |

目前 `Green` 是唯一公开的单色显示规范。

## 为什么需要独立的 Monochrome 体系

单色显示设备和全彩设备在视觉表达上有明显差异：

- 只能使用一个色相，不能依赖红色、蓝色或多色标签表达状态
- 阴影在透明 AR 显示中可读性较差，更适合使用清晰描边和轻量填充
- 内容必须控制在舒适视野内，超过高度上限后应进入滚动
- 错误态、弱提示和高亮态都必须在同一色相内完成表达

## 当前变体：Green

`Green` 是 AIUI 当前启用的单色显示视觉规范，面向 RokidGlasses1 和 RokidGlasses2 的单绿色显示硬件。

这套规范的核心约束是：设备只能在纯黑背景上渲染单一绿色发光通道，因此界面不能依赖第二种颜色来表达状态，而要用透明度、描边、留白和排版层级构建完整的 HUD 视觉语言。

## Green 的设计约束

- **单绿色显示**：只有绿色通道可用，不能使用红色、蓝色或其他色相
- **透明显示背景**：黑色底层会与真实世界叠加，界面是“悬浮”而不是“遮挡”
- **舒适视野约束**：内容宽度固定，高度受限，超出后优先滚动
- **无多色语义**：错误态、弱提示和高亮态都必须通过同一绿色体系表达

## Green 的核心特征

- **单色透明度阶梯**：通过 100% / 60% / 40% / 8% 四档绿色透明度建立层级
- **描边优于阴影**：结构与强调依赖边框、填充和留白，而不是模糊阴影
- **卡片式浮层**：所有内容都以固定宽度卡片面板承载
- **Token 优先**：颜色、排版、间距、边框和组件参数都应通过 Design Tokens 定义
- **错误态仍保持绿色**：错误提示不切换到红色，而是用更弱的边框和更浅的填充表达

## 画布与布局

| Token | 含义 | 值 |
| --- | --- | --- |
| `app-width` | 默认画布宽度 | `448px` |
| `height-min` | 默认最小高度 | `120px` |
| `height-max` | 默认最大高度 | `352px` |

推荐把界面实现为固定宽度的卡片面板，超过最大高度后进入滚动布局，而不是继续向下无边界扩展。

## 颜色系统

| Token | 用途 | 值 |
| --- | --- | --- |
| `primary` | 主强调色、标题、关键数据 | `#40ff5e` |
| `primary-60` | 次级文字、默认边框 | `rgba(64,255,94,0.6)` |
| `primary-40` | 轻量填充、高亮 surface、弱分隔 | `rgba(64,255,94,0.4)` |
| `primary-08` | 输入框和错误容器背景 | `rgba(64,255,94,0.08)` |
| `background` | 页面背景 | `#000000` |
| `surface` | 卡片和容器背景 | `#000000` |

在这套系统里，颜色不是“多色板”，而是一条单绿色的明度与透明度阶梯。

## 排版基线

| Token | 字体族 | 大小 | 字重 | 用途 |
| --- | --- | --- | --- | --- |
| `display` | `monospace` | `24px` | `700` | 面板标题、重点标题 |
| `heading` | `monospace` | `18px` | `700` | section 标题 |
| `body` | `sans-serif` | `15px` | `400` | 正文 |
| `body-sm` | `sans-serif` | `13px` | `400` | 紧凑信息、表格内容 |
| `label` | `sans-serif` | `13px` | `600` | 按钮文字、字段标签 |
| `caption` | `sans-serif` | `11px` | `400` | 辅助信息、时间戳 |

推荐使用 monospace 承载标题，用 sans-serif 承载正文，从而在单色显示里进一步放大层级差异。

## 间距、圆角与边框

### 间距

- `xs`: `4px`
- `sm`: `8px`
- `md`: `12px`
- `lg`: `18px`
- `xl`: `24px`
- `xxl`: `32px`

### 圆角

- `sm`: `12px`
- `md`: `12px`
- `full`: `9999px`

### 边框宽度

- `thin`: `1px`
- `default`: `2px`
- `strong`: `4px`

这套系统偏向“轻填充 + 明确描边 + 稳定留白”，比依赖阴影更适合透明 AR 显示。

## 组件基线

| 组件 | 默认表达 |
| --- | --- |
| `card` | 黑色 surface + 2px 绿色默认描边 + 12px 圆角 |
| `card-highlight` | 绿色 40% 填充 + 绿色强调描边 |
| `text-input` | 8% 绿色背景 + 1px 默认描边 |
| `textarea` | 与 `text-input` 同构，用于长文本输入 |
| `button` | 以描边和标签文字表达强调，而不是实体填充按钮 |
| `error-state` | 8% 绿色填充 + 40% 绿色边框 + 主绿色文字 |
| `chart-container` | 复用 `card` 的容器表达方式 |

## Do

- 用卡片和描边建立结构
- 用绿色透明度阶梯建立层级
- 超过高度上限时切换到滚动布局
- 优先复用 token，而不是在页面里写死样式
- 用边框粗细、surface 层级和字重表达强调

## Don't

- 不要引入第二种颜色
- 不要依赖模糊阴影表达层级
- 不要把错误态做成红色
- 不要让内容无限扩展超出舒适视野
- 不要依赖 hover 作为主要交互反馈

Visual Design · single-green display

Rokid AIUI — green HUD over the real world

A transparent-display HUD aesthetic constrained to single-green AR glasses (RokidGlasses1 / RokidGlasses2), whose hardware renders only one luminous green channel over pure black. The entire interface is one hue at four opacity tiers — outlines, not shadows; translucent fills; stable whitespace. A host-driven Design Token system tuned for legibility on monochrome-green hardware.

01 — Foundations

Color Palette

A single green at four opacities over pure black — dictated by the single-green display of RokidGlasses1 / RokidGlasses2. No red, blue, or secondary hue is physically available. Every emphasis decision is an opacity step on the same hue, never a different color.

02 — Foundations

Typography

Titles use the monospace stack — the fixed-width numerals read like a heads-up terminal over the real world. Body copy and UI labels stay on the proportional sans-serif (system/Android default) stack. No named typeface is required. Hierarchy is by weight and size, not color — since the whole palette is green, size and weight must carry the structure.

03 — Components

Buttons & Controls

Emphasis is by border, not a filled colored background. The native button component carries built-in touch feedback for gaze + temple-tap interaction. On a single-green display, a full-green fill is the strongest possible affordance.

button · default

Outlined accent — green border, green label, 12px radius. The everyday action.

button · strong fill

Full-green fill with black label — the highest-contrast "commit" action.

button · strong border

4px strong border — key interactive / active state.

status pill · full radius● online

Pill chip — full radius reserved for status and control affordances.

04 — Components

Cards & Surfaces

The foundational surface. Nearly every interface is a vertical stack of outlined cards inside the 448px canvas. Depth comes from outline weight and green opacity — never shadow.

card · default

Black fill + 2px green-60% outline, 12px corners, 12px padding. The base container every agent interface is built from.

card-highlight

Green-40% fill + full-green accent border — the "raised" surface, used in lieu of shadow for emphasized cards and demo blocks.

error-state

Green-08% fill + muted green-40% border. Stays green — error is expressed by container treatment, never a red hue.

divider / list-row

1px green-40% rule between content blocks. List rows separate via muted dividers inside a scroll-view.

08 — Foundations

Elevation & Depth

Outline and opacity, not shadow. The spec is explicit: light fills, clear outlines, and stable whitespace suit a transparent AR display better than shadows. There is no blurred drop-shadow vocabulary.

0 — Recessed field
8% green fill + 1px default outline. Inputs, textareas, error hints.
1 — Card
Black + 2px green-60% outline. Default cards and chart containers.
2 — Highlighted card
Green-40% fill + full-green accent outline. Emphasized cards, demo blocks.
3 — Accent emphasis
Full-green border thickened to 4px. Key interactive states, active rows.
09 — Device constraint

Single-Green Display (RokidGlasses1 / 2)

A fixed 448 x 352 canvas, form-factor-locked design authored for single-green AR glasses. The host theme defines the geometry; the app fills it. Content that exceeds the max height enters a scroll-view rather than resizing. Only the green channel renders, so meaning must never rely on red/blue hue cues.
